Flash File Format SDK – README
January 28, 2000

The Macromedia Flash File Format SDK provides a set of tools for developers to write Macromedia Flash 4 SWF files. The File Format SDK is comprised of a low level interface and a high level interface, both of which write SWF files, but are intended to be used in different ways.

Installation and Executing

The project is provided ready to compile on a Windows systems. A Visual C++ project file is provided.

  1. Copy the Source directory to your hard disk. 
  2. Open the Workspace file, swfproduce.dsw with Visual C++. 
  3. Build All 
  4. Now you should be able to run swfproduce.exe directly from the environment. 

Note that if you run from the command line, the bm128.jpg and audio.wav files need to be in the current directory. This should not be a problem if you run from the Visual C++ IDE.

Execution will briefly bring up a console app which should run and quickly complete. The app will write a series of sample SWF files to the local directory. You can then view the sample .SWF files and the example .cpp files that created them, which have the same base name. For instant SWF gratification, try looking HFExample.swf.

There will be 2 sets of examples written to your disk:

  • HFExample*, examples created with the high level manager. 
  • FExample*, examples created with the low level manager. 

Documentation

The SWF File Format Documentation is provided in both HTML and MS Word 97 formats. Documentation is provided for:

  • The SWF File Format, explaining how the file format works.
  • The SWF manager, which gives programmers an API with which to write SWF files.

Additional notes about the documentation can be found in the Documentation subdirectory.

The first 2 subdirectories below Flash_File_Format_SDK are HTML and MSWord. The entire documentation is echoed in both formats. (Except the API_Docs which are only available in HTML format, but the HTML files are in the MSWord tree for completeness.)

The SWF_File_Format_Docs directory contains an explanation of the Macromedia SWF file format. The file "intro.doc/html" provides a basic introduction and links to the individual SWF topics.

The SWF_Manager_Docs directory contains the "SDK.doc/html" file which describes the basics of using the high and low level manager to create SWF files. The subdirectory API_Docs is a reference (in html) to the high and low level API. The *_files subdirectories contain images for the html docs, and should be ignored.

Using the SDK in other programs

  1. Add all the File Format SDK files to your project, including the subdirectories PCM and ZLib-1.1.3. 
  2. Remove all example files HFExample*.* and FExample*.*
  3. Remove swfproduce.cpp (contains main() ) 
  4. Include the HF3SDK header in your files that use the high level SDK, and the F3SDK header for files that use the low level SDK. 

The Low Level Interface

The low level interface can write SWF files of any type. Every Flash 4 SWF type (covered in the SWF Format Documentation) is represented by a class in the low level manager. Classes that begin with ‘F’ are low level classes. Examples of the low level interface are in the FExample* files. You would use the low level interface if you understand SWF and wish to write SWF tags directly, or if you are trying to write unusual SWF files.

Documentation for the low level classes is provided in hyper linked html, located in the "API_Documentation" subdirectory. It is in Java doc format. 

The low level manager contains many examples, for which both the SWF file and the .cpp file have the FExample* prefix.

To examine some sample code, the best place to start is FExampleBitsLossless.cpp. The lossless bitmap example has the most detailed documentation. 

The High Level Interface

The high level interface is used to write SWF in a more natural way. The developer can add Rectangle, Circle, Bitmap, EditText, and other primitive objects to Frames in a Movie. The Movie can then be serialized to disk. It's an easy, fast way to write SWF.

High level classes begin with ‘HF’. Examples of the high level interface are in the HFExample* files.

Documentation for the high level classes is provided in hyper linked html, located in the "API_Documentation" subdirectory. 

To review some sample high level code, the best place to start is HFCircle.cpp, since it this is the example easiest to understand. HFExample.cpp is also worthwhile to examine; while much larger, it demonstrates all of the features included in the high level manager.
